The Investors API lets your application register new participants in the KitaFund SPCX Market IPO access program. Every investor record carries a `status` field that controls whether the investor may participate in allocations and deposits.

---

## POST /api/investors

Register a new investor account. This is the entry point for all new users of the platform. On success the API returns `201 Created` with the full investor object.

**Endpoint:** `POST /api/investors`

### Request Body

<ParamField body="fullName" type="string" required>
  The investor's full legal name. Leading and trailing whitespace is stripped before storage.
</ParamField>

<ParamField body="email" type="string" required>
  The investor's email address. Must be a valid email format. The value is normalised to lowercase. Returns `409 Conflict` if the email is already registered.
</ParamField>

### Response Fields

<ResponseField name="id" type="number">
  Auto-assigned unique integer identifier for the investor record.
</ResponseField>

<ResponseField name="fullName" type="string">
  The investor's full name as stored.
</ResponseField>

<ResponseField name="email" type="string">
  Normalised lowercase email address.
</ResponseField>

<ResponseField name="status" type="string">
  Initial account status. Always `"pending"` for newly created accounts.
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO 8601 timestamp of when the account was created, e.g. `"2025-11-03T14:22:00.000Z"`.
</ResponseField>

### Code Example

<CodeGroup>
```bash curl
curl -X POST https://spcxipo.live/api/investors \
  -H "Content-Type: application/json" \
  -d '{
    "fullName": "Ada Lovelace",
    "email": "ada.lovelace@example.com"
  }'
```

```javascript JavaScript
const response = await fetch("https://spcxipo.live/api/investors", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    fullName: "Ada Lovelace",
    email: "ada.lovelace@example.com",
  }),
});

const investor = await response.json();
```
</CodeGroup>

### Success Response

```json
{
  "id": 42,
  "fullName": "Ada Lovelace",
  "email": "ada.lovelace@example.com",
  "status": "pending",
  "createdAt": "2025-11-03T14:22:00.000Z"
}
```

<Note>
  All new accounts start with `status: "pending"`. The KitaFund compliance team reviews each application before granting access to deposit and allocation features. To check approval state, call [POST /api/auth/signin](/api/auth) with the investor's email.
</Note>

### Error Responses

| Status | Error message | Cause |
|--------|--------------|-------|
| `400` | `"Invalid input. Full name and email are required."` | One or both required fields are missing or malformed. |
| `409` | `"This email is already registered."` | An investor account already exists for this email. Use `POST /api/auth/signin` to sign in instead. |
| `500` | `"Something went wrong. Please try again."` | Unexpected server error. |
